Що таке JavaScript?

JavaScript була створена для того, щоб "оживати сторінки".

Програми на цій мові називаються скриптами. Їх можна писати прямо на HTML-сторінці і вони автоматично компілюються при завантаженні сторінки.

Для запуску JS не потрібні спеціальні компілятори чи спеціальна підготовка. У цьому плані JS дуже відрізняється від інших мов програмування - Java.

Чому цю мову називають JavaScript?

Коли створили JS, тоді вона мала назву LiveScript. В той час була досить популярною мова Java, тому задля популярності було названо JavaScript, типу "молодша систра" Java

Сьогодні JavaScript виконується не тільки у браузері, але і на серверах, або на будь-якому пристрої де є - рушій JavaScript.

Браузер має вбудований рушій, який деколи називають "віртуальною машиною JavaScript".

  • V8 - Chrome та Opera
  • SpiderMonkey

Ці віртуальні машини потрібно запамʼятати

Як працюють рушії?
Рушій є складним, але принцип роботи простий.
  • Рушій (вбудований, якщо це браузер) читає ("розбирає") скрипт.
  • Потім він перетворую скрипт в байт код.
  • І потім байт код виконується, і дуже швидко.
Рушій застосовує оптимізації на кожеому етапі процесу. Він навіть слідкує за скомпільваним скриптом під час його виконання, аналізує дані, які проходять через скрипт, і оптимізує байт код на основі цих знань.

Що може вбудований в браузер JS

Сучасний JavaScript - це безпечна мова програмування. Вона не дає низькорівневого доступу до памʼяті чи процесора.

Можливості JS залежать від оточення де її застосують. Наприклад, Node.js підтримує функції, які дозвояють читати/записувати різні файли, здійснювати запити, та інше.

Вбудований в браузер JS може робити все що повʼязано з маніпуляцією веб-сторінки

Наприклад, JS може:

  • Додавати новий HTML-код на сторінку, змінбвати існуючий вміст та змінювати стилі.
  • Реагувати на дії користувача.
  • Відправляти запити.
  • Отримати і надсилати куки.

Що НЕ може JavaScript

Можливості JavaScript у браузері обмежані з метою безпеки користувача. Мета полягає в тому, щоб заборонити небезпечним сторінкам доступ до приватної інформації.

Приклади таких обмежень:

  • JavaScript на веб-сторінці не може читати/записувати довільні файли на жорстому диску.

    • Сучасні брайзери дозвояють працювати з файлами, але доступ до них є обмежаний та надається тільки тоді, коли користувач виконав відповідні дії, наприклад, претянув файл у вікно тегу

    <іnput>

  • Різні вкладки не знають один одного. Іноді це можливо, коли використовується JavaScript для того щоб відкрити інше.
  • JavaScript може легко спілкуватися через мережу з сервером, від якого отримана поточна сторінка. Але її здатність отримувати дані з інших сайтів/доменів обмежена.

Таких обмежень немає, якщо JavaScript використовується за межами браузера, наприклад, на сервері.

Що робить JavaScript унікальною?

Є як мінімум три особливості JS:

  • Цілкова інтеграція з HTML/CSS.
  • Прості речі робляться просто.
  • Підтримується усіма сучасними браузерами

JavaScript – це єдина браузерна технологія, яка суміщає ці три речі.

Мови "над" JavaScript

Синтаксис JavaScript не задовольняє потреби кожного. Різні люди хочуть різних функцій. Цього слід очікувати, тому що проекти і вимоги різні для кожного.

Останнім часом зʼявилося безліч нових мов, які транспілюються в JavaScript до того, як використовуються в браузері.

Сучасні інструменти роблять транспіляцію дуже швидкою і прозорою, дозволяючи розробникам писати код іншою мовою і автоматично конвертувати його “під капотом”.

Приклдаи таких мов:

  • CoffeeScript — це “синтаксичний цукор” поверх JavaScript. Вона вводить більш короткий синтаксис, що дозволяє нам писати більш чіткий і точний код. Зазвичай, це до вподоби програмістам на Ruby.
  • TypeScript зосереджена на додаванні “строгої типізації даних”, щоб спростити розробку і підтримку складних систем. Розробляється у Microsoft.
  • Flow також додає типізацію даних, але іншим способом. Розробляється компанією Facebook.
  • Та інші

Підсумки
  • Мова JavaScript спочатку була створена лише як мова для браузера, але зараз її також використовують в інших середовищах.
  • Сьогодні, JavaScript позиціонується як найбільш поширена мова для браузера, яка повністю інтегрована з HTML/CSS.
  • Є багато мов які “транспілюються” в JavaScript і надають певні функції. Рекомендується переглянути їх, принаймні мигцем, після освоєння JavaScript.

Специфікація

Специфікація ECMA-262 містить найглибшу, найдетальнішу й найбільш формалізовану інформацію про JavaScript. Фактично, ця специфікація визначає мову.

Довідники

MDN (Mozilla) JavaScript Reference — це головний довідник з прикладами та іншою інформацією. Він чудово підходить для детального вивчення окремих функцій, методів тощо.

Редактори коду
  1. IDE
  2. Легкі редактори

Є два види редакторів коду: IDE та легкі редактори.

IDE

Термін IDE (Інтегроване середовище позробки) означає потужний редактор з багатьма можливостями, що зазвичай працює з "цілим проєктом". Як вже зрозуміло що це ціле середовище розробки, де не потрібно перикатись на різні додатки

IDE завантажує проєкт, дозволяє переключатис між файлами, надає можливість автозаповнення, інтегрується з системоє контролю версій.
